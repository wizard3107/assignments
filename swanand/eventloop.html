Event Loop

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks

Call Stack

Whenever you call a function in javascript all the instruction which the function contain get loaded into the stack,then javascript execute the function by pooping it out from the stack.

Web API

Whenever javascript encounter's a piece of code that will take long time to execute,it hand over the that piece code to web api for particular time period.Web Api is just a code holder which hold the particular code for the particular time period and then send the piece of code to the call back queue.

Callback Queue

It is another data structure that facitilate javascript to add next code to execute.It works on the First in First out method,Whenever JS finds a piece on code which take some time it handover it to the web api and after particular time period web api sent that code to callback queue.Then callback queue send the piece of code in the callstack only when callstack is empty.
